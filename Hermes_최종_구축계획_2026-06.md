## 0. 핵심 원칙 — 헤르메스는 "지휘관"이다

이 계획 전체를 관통하는 단 하나의 원칙:

> **헤르메스(특히 공명)는 직접 일하는 일꾼이 아니라 지휘관이다.**
> 무거운 작업은 직접 하지 않고 분해해서 전문 CLI에 위임한다.

이게 비용 구조를 통째로 바꾼다:

- 지휘관은 "추론 천재"일 필요가 없고 **도구 호출·계획·분배를 잘하는 모델**이면 충분하다.
- 무거운 추론(코딩·리팩토링·테스트)은 **이미 구독 중인 Claude Code / Codex / Gemini CLI에 위임** → 토큰당 과금이 아닌 정액으로 처리.
- 따라서 OpenRouter로 비싼 `opus-4.8`를 부를 일이 거의 사라진다 → **개인 지갑에 압도적으로 유리.**

| | 실행자 모델(틀림) | 지휘관 모델(채택) |
|---|---|---|
| 헤르메스 본체 | 직접 코딩·추론 | 의도분석 → 분해 → 위임 → 취합·검증·보고 |
| 어려운 코딩 | OpenRouter `opus-4.8` (토큰과금, 지연 32s) | **Claude Code 위임** (구독제) |
| 테스트·격리실행 | 헤르메스 직접 | **Codex CLI 위임** |
| 멀티모달·대안 | 헤르메스 직접 | **Gemini CLI 위임** |
| 본체 모델 | opus까지 필요 | 툴사용 최강 저가모델로 충분 |

> ⚠️ **선결 조건**: 현재 설치된 Hermes 버전이 외부 CLI 오케스트레이션(Cross-CLI / PTY 백그라운드 위임, docx Issue #413)을 지원하는지 먼저 확인. 미지원이면 내부 `delegate_task` 하위 에이전트만으로 시작.

## 2. 최종 모델 라우팅

> **원칙(Codex 검증)**: 폴백 = 장애 대응, 승급 = 명시적 선택. 품질 낮다고 비싼 모델로 자동 승격 금지.
> opus/gpt-5.5/sonnet은 지연 32~108초라 자동 폴백에 넣으면 지갑+체감 둘 다 망함.

### 2-1. 지휘관 본체 모델

| 역할 | 권장 모델 | 근거(데이터) |
|---|---|---|
| **헤르메스 지휘관 (공명)** | `google/gemini-3.5-flash` (또는 `deepseek-v4-pro`) | "에이전트/툴 사용은 Pro 능가, Flash 비용" — 오케스트레이션 최적 |
| **방통 (홈·잡담)** | `deepseek/deepseek-v4-flash` | 가성비 204, 지연 1.4s |

### 2-2. 무거운 작업은 위임 (토큰과금 회피)

| 작업 | 위임 대상 | 근거 |
|---|---|---|
| 코드 구현·리팩토링 | **Claude Code (PTY)** | 구독제, SWE 88.6 그대로 |
| 단위테스트·격리실행 | **Codex (PTY)** | 빠른 격리 테스트 |
| 멀티모달·대안검토 | **Gemini CLI** | 영상·이미지 |
| 내부 병렬 하위작업 | delegate_task 하위에이전트 (`deepseek-v4-pro`) | 격리 샌드박스 병렬 |

→ OpenRouter `opus` premium 경로는 **거의 폐기**. 그 일은 Claude Code가 구독으로 처리.

### 2-3. 직접 처리 시 작업등급 라우팅(위임 불가한 경량 작업용)

| 작업 등급 | 모델 |
|---|---|
| 잡담·일정·텔레그램 즉답 | `deepseek-v4-flash` |
| 구조화 답변·요약 | `deepseek-v4-pro` |
| 긴 문서 요약·다중 비교 | `gemini-3.5-flash` |
| (예외) 최종 설계·실패비용 큰 작업 | `opus-4.8` — `/premium` 명령 필수, 가급적 Claude Code 위임 우선 |

## 3. config.yaml (지금 반영)

```yaml
model:
  provider: openrouter
  default: google/gemini-3.5-flash      # 지휘관 기본 (툴사용 최강·저가)
                                        # 방통 세션은 gatekeeper가 flash로 override

provider_routing:
  sort: price
  require_parameters: true
  data_collection: deny                 # ★ 프롬프트 학습 거부 (보안)
  ignore: ["OpenRouter:free", "owl-alpha"]   # 로깅 위험 회피

fallback_providers:                      # 장애 폴백만 — 자동 고급승급 아님
  - { provider: openrouter, model: deepseek/deepseek-v4-pro }
  - { provider: openrouter, model: deepseek/deepseek-v4-flash }

auxiliary:                               # 보조 작업은 전부 최저가 고정
  compression:      { provider: openrouter, model: deepseek/deepseek-v4-flash }
  title_generation: { provider: openrouter, model: deepseek/deepseek-v4-flash }
  vision:           { provider: openrouter, model: google/gemini-3.5-flash }

delegation:
  enabled: true
  subagent_model: deepseek/deepseek-v4-pro    # premium subagent 금지
  # 외부 CLI 하이브리드 — 지휘관이 PTY 백그라운드로 호출 (버전 지원 확인 필요)
  external_clis:
    - { name: claude-code, role: "코드베이스 리팩토링·구현", cmd: "claude" }
    - { name: codex,       role: "단위테스트·격리실행",      cmd: "codex exec --skip-git-repo-check" }
    - { name: gemini,      role: "멀티모달·대안검토",        cmd: "gemini --skip-trust -p" }
```

> Hermes 버전별로 외부 CLI 통합 방식(PTY/스킬/MCP)·키 이름이 다름. 위는 개념 스키마이며 `hermes config` 실제 스키마로 맞출 것.

## 4. ⭐ 캐싱 함정 — SOUL.md 통째 교체가 비용을 올린다

내관이 `SOUL.md`를 통째로 갈아끼우면 시스템 프롬프트 prefix가 바뀌어 **OpenRouter 프리픽스 캐시가 매번 깨짐** → 인격 전환마다 풀 토큰 재과금.

**해결 — prefix 고정 + persona delta만 교체:**

```text
[고정 공통 시스템 규칙]          ← 절대 안 바뀜 (캐시 히트)
[공유 스킬/메모리/도구 규칙]
[persona block: 공명 or 방통]    ← 여기 작은 블록만 교체
[현재 요청]
```

즉 SOUL.md 전체가 아니라 **인격 차이 부분만 작은 블록으로** 둔다.

## 5. 비용 가드 (개인 지갑)

- OpenRouter 월 cap: **1차 $10 / 상한 $20** (provider 대시보드 cap 필수, Hermes 내부설정만 믿지 말 것).
- API 키 분리: `gm-main`(일상) / `gm-premium`(opus용, 더 낮은 한도).
- **`max_turns` 150 → 100** 축소 (텔레그램 대화는 꼬리가 길어 비용·오염 동반 증가).
- 매일 OpenRouter 잔액(`/api/v1/credits`)을 텔레그램으로 자동 리포트 (cron). management key 필요.
- 무거운 작업 위임으로 토큰 소비 자체를 줄이는 게 가장 큰 비용 절감.

## 6. 공명 SOUL.md — 지휘 원칙 주입

```markdown
# 행동 원칙 (지휘관)
- 너는 직접 손을 대는 일꾼이 아니라 지휘관(제갈량)이다.
- 무거운 작업은 직접 하지 말고 분해해서 전문가에게 위임한다:
  · 코드 구현/리팩토링 → Claude Code 호출
  · 테스트/격리 실행 → Codex 호출
  · 멀티모달/대안 검토 → Gemini 호출
- 네 일은: 의도 파악 → 작업 분해(DAG) → 위임 → 결과 검증·취합·보고.
- 위임 전 항상 한 줄 실행계획을 먼저 말한다.
```

## 7. 이중 인격 운영

### 7-1. 역할 분담 매트릭스

| 구분 | 공명(워크) | 방통(홈) |
|---|---|---|
| 전장 | 터미널/WSL, 코딩, 보고서 | 텔레그램/모바일, 일상 |
| 권장 모델 | gemini-3.5-flash(지휘) + 위임 | deepseek-v4-flash(즉답) |
| 말투/톤 | 격식체, 병법 비유, 결론 위주 | 구어체, 익살, 공감 |
| 출력 형식 | 구조화 마크다운·코드블록 | 짧은 메시지·이모지 |
| 핵심 가치 | 무결성·정확도(Zero-defect) | 민첩성·편의(Quick-win) |

### 7-2. 하루 운영 시나리오

| 시간 | 채널/인격 | 내용 |

| 08:30 | 텔레그램/방통 | 아침 브리핑(날씨·일정·미완료 메모) |
| 10:00 | 터미널/공명 | 업무 지휘 — 작업 분해 후 Claude Code에 구현 위임 |
| 14:00 | 터미널/공명 | 복잡 에러 → Codex로 테스트 위임, 결과 취합·검증 |
| 19:00 | 텔레그램/방통 | 퇴근길 — 오늘 결과 가볍게 리뷰 |
| 23:00 | 텔레그램/방통 | 하루 마감 — 메모리 정리·요약 |

### 7-3. 메모리·스킬 (공유 뇌 오염 방지)

- **USER.md (불변)**: 기본 정보, 직업, 기술 스택, 금기, 두 인격 정의. (1375자 한도)
- **MEMORY.md (학습된 사실)**: `[WORK]`/`[LIFE]` 머리말로 인격 구분. (2200자 한도)
- **스킬 80+개**:
  - 인격별 우선 스킬을 persona 블록에 명시 (공명: File/Git/Code, 방통: Calendar/Search).
  - 핵심 스킬 pin: `Memory_Update`, `Context_Summarizer`는 `hermes curator pin`으로 자동삭제 방지.
  - 주 1회 미사용 스킬 정리 → 추론 비용 절감.
- **편의**: alias `hm`(공명 직행)·`bt`(방통 직행)으로 내관 단계 생략. 텔레그램에 "입력 중…" 상태 표시.

## 8. 리스크 Top 5 + 완화 (Codex)

| 리스크 | 완화책 |
|---|---|
| SOUL.md 교체 race condition (CLI·게이트웨이 동시 접근→반쪽파일) | 임시파일 쓰고 **atomic rename + 파일 lock** |
| 공유 메모리 오염 (방통 농담이 공명 컨텍스트에) | memory에 `persona/domain` 태그, 검색 가중치 |
| 스킬 오염 | 인격별 skill allowlist/denylist |
| gatekeeper 단일 장애점 (SOUL 교체 실패→오인격) | 실패 시 `flash + 중립 SOUL`로 degrade, checksum 로그 |
| 캐시 무효화 (인격 전환마다 cache miss) | §4 공통 prefix 고정 + persona delta |
| (지휘관 추가) 위임 실패·결과 미검증 | 위임 결과는 항상 지휘관이 검증 후 취합. 하위 1개부터 테스트 |
